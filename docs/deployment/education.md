# 教育场景部署

Ditto WebOS 特别适合教育场景，本页面介绍学校/培训机构的部署指南。

## 适用场景

| 场景 | 描述 |
|------|------|
| 课堂教学 | 教师演示软件，学生跟随操作 |
| 编程教学 | 学生在 Ditto 上开发应用 |
| 远程教育 | 通过浏览器访问虚拟桌面 |
| 考试环境 | 安全隔离的考试应用 |

## 部署方案

### 方案一：学校服务器部署

在学校机房服务器部署 Ditto，学生通过内网访问：

```
┌─────────────┐
│ 学校服务器   │ ← Ditto Server + Nginx
└─────────────┘
       ↓ 内网
┌─────────────┐
│ 学生电脑    │ ← Chrome 80+ 即可访问
└─────────────┘
```

### 方案二：云服务器部署

租用云服务器，学生通过公网访问：

- 优点：支持远程访问、多地协作
- 缺点：需要公网带宽、HTTPS 证书

### 方案三：树莓派部署

Ditto 支持在树莓派上运行：

```bash
# 在树莓派上安装 Bun
curl -fsSL https://bun.sh/install | bash

# 启动 Ditto
bun run start
```

- 优点：成本低、便携
- 注意：需设置动画档位为「关闭」或「微妙」

## 性能优化

针对低性能设备（老旧 PC、树莓派）：

1. **动画档位**：设置为「关闭」或「微妙」
2. **Widget throttling**：4fps RAF 限制
3. **输入 debounce**：150ms 输入延迟
4. **PWA 缓存**：减少网络请求

## 教师管理

Ditto 支持教师账号管理：

- 分发预装应用
- 监控学生应用使用
- 远程协助（通过 IPC）
